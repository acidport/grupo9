ESTANDARIZACION Y EJEMPLOS DE UNA BASE DE DATOS

        Para proceder a estandarizar la base de datos en cuanto sus sentencias se usará un estándar propio el cual ayudará 
        a la lectura y manipulación de los datos de esta forma:
        Tablas (TBL_): 
        Dentro de las tablas existen diferencias de acuerdo a su uso para este caso mencionare tres Referencial, 
        Transaccional y de unión para relaciones muchos a muchos su forma de estandarizarlo al escribirlas son 
        TBL_ donde indica que es una tabla, el siguiente carácter indicara el tipo de dicha tabla 

        (R) referencia 
        (T) transaccional  
        (U) para unión 

        seguido del nombre de la tabla en singular y descriptivo.
        Ejemplo
        Para ventas: TBL_TVENTA
        Para usuarios: TBL_RUSUARIO
        Para Perfiles de usuario: TBL_RPERFIL
        Para usuarios que tienen un perfil: TBL_UUSUARIO_PERFIL.

        Nombres de campos en una tabla:
        Sera iniciado con los tres primeros caracteres que indican la tabla seguidos de un guion bajo, seguidos del tipo de dato
        y seguidos con el nombre del campo en singular.
         
        Ejemplo para la tabla TBL_RUSUARIO sus campos empezaran USU_
        Para indicar el tipo de dato que almacenara dicho campo se asignaran el siguiente carácter así:
        N numérico
        C carácter o String
        O booleano o bit
        D fecha
        Si el campo es una llave primaria seguirá después del carácter (ID)
        Así quedaría la llave primaria de la tabla TBL_RUSUARIO:
        USU_NID
        Para el campo nombre
        USU_CPRIMER_NOMBRE
        Para saber si es activo o no
        USU_OESTADO
        Para saber el ingreso de una venta
        VEN_DFECHA_VENTA
        Si la tabla tiene una llave foránea está se deberá llamar igual a como este escrita en la tabla origen así por ejemplo
        en la tabla transaccional venta 
        TBL_TVENTA
        el campo del usuario que realiza la venta se llamara USU_NID, siendo está foránea, esto es importante ya que ahorrara 
        pasos si más adelante se quiere realizar un OLAP.
