Lecturas

LDD

Que es el LDD en bases de datos.

Un lenguaje de base de datos o lenguaje de definición de datos (Data Definition Language, DDL por sus siglas en inglés) es un lenguaje proporcionado por el sistema de gestión de base de datos que permite a los programadores de la misma llevar a cabo las tareas de definición de las estructuras que almacenarán los datos así como de los procedimientos o funciones que permitan consultarlos.

LCD

Que es el LCD en bases de datos.
Es un lenguaje proporcionado por el Sistema de Gestión de Base de Datos que incluye una serie de comandos SQL que permiten al administrador controlar el acceso a los datos contenidos en la Base de Datos.

    Sistemas gestores de base de datos que utilizan SQL:
    SQL Server 
    Oracle
    MySql
    PostgreSQL
    DDL:
    - Es la parte del SQL dedicada a la definición de la base de datos.
    - Permite crear índices, definir reglas de validación de datos, relaciones etre tablas, etc.
    DDL (Lenguaje de definición de datos)
    DML (Lenguaje de manipulación de datos)
    DCL (Lenguaje de control de datos)
    Lenguaje estructurado de consulta (Structured Query Language)
    Lenguaje utilizado para definir, controlar y accerder a los datos almacenados en una BD relacional.

    Es un lenguaje universal que se emplea en cualquier sistema gestor de datos relacional.
    En SQL Server la versión de SQL que se utiliza se llama TRANSACT - SQL.
    DML:
    Se compone de las instrucciones para el manejo de los datos es decir para insertar, modificar, eliminar, y seleccionar datos
    DCL:
    Se componen de instrucciones que permiten ejercer un control sobre los datos. Es responsable de la gestión de transacciones (COMMIT/ROLLBACK)..

DML

El lenguaje de manipulación de datos (DML) es un vocabulario utilizado para recuperar y trabajar con datos en SQL. Existen declaraciones las cuales se podrán realizar con cada una de ellas agregar, modificar, consultar o eliminar datos de una base de datos de SQL Server. 
        Las sentencias usadas son 4:

        *Select: 
            Con este comando se extrae la data de las tablas. 
            Ejemplo: select * from tabla
        *Insert: 
            Con este comando se inserta datos en una tabla.
            Ejemplo: insert into tabla (campo1, campo2, campon) values(valor1, valor2, valorn)
        *Update
            Con este comando actualiza datos de una tabla.
            Ejemplo: Update tabla set campo1=valor1, campo2=valor2, campon=valorn
        *Delete
            Con este comando se podra eliminar datos de una tabla.
            Ejemplo: Delete from tabla where campo1=valor1

.

ESTANDARIZACION Y EJEMPLOS DE UNA BASE DE DATOS

        Para proceder a estandarizar la base de datos en cuanto sus sentencias se usará un estándar propio el cual ayudará a la lectura y manipulación de los datos de esta forma:
        Tablas (TBL_): 
        Dentro de las tablas existen diferencias de acuerdo a su uso para este caso mencionare tres Referencial, Transaccional y de unión para relaciones muchos a muchos
        Su forma de estandarizarlo al escribirlas son TBL_ donde indica que es una tabla, el siguiente carácter indicara el tipo de dicha tabla 

        (R) referencia 
        (T) transaccional  
        (U) para unión 

        seguido del nombre de la tabla en singular y descriptivo.
        Ejemplo
        Para ventas: TBL_TVENTA
        Para usuarios: TBL_RUSUARIO
        Para Perfiles de usuario: TBL_RPERFIL
        Para usuarios que tienen un perfil: TBL_UUSUARIO_PERFIL.

        Nombres de campos en una tabla:
        Sera iniciado con los tres primeros caracteres que indican la tabla seguidos de un guion bajo, seguidos del tipo de dato y seguidos con el nombre del campo en singular.
         
        Ejemplo para la tabla TBL_RUSUARIO sus campos empezaran USU_
        Para indicar el tipo de dato que almacenara dicho campo se asignaran el siguiente carácter así:
        N numérico
        C carácter o String
        O booleano o bit
        D fecha
        Si el campo es una llave primaria seguirá después del carácter (ID)
        Así quedaría la llave primaria de la tabla TBL_RUSUARIO:
        USU_NID
        Para el campo nombre
        USU_CPRIMER_NOMBRE
        Para saber si es activo o no
        USU_OESTADO
        Para saber el ingreso de una venta
        VEN_DFECHA_VENTA
        Si la tabla tiene una llave foránea está se deberá llamar igual a como este escrita en la tabla origen así por ejemplo en la tabla transaccional venta 
        TBL_TVENTA el campo del usuario que realiza la venta se llamara USU_NID, siendo está foránea, esto es importante ya que ahorrara pasos si más adelante se quiere realizar un OLAP.
